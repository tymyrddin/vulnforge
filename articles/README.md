# Articles

Three pieces on the idea behind vulnforge, in reading order. They
explain what the tool is for and how it works without assuming a security or AI
background.

1. [The model is not the system](model-is-not-system.md). Why serious AI work is
   becoming an arrangement of parts rather than one clever model, and where vulnforge
   sits as a worked example.
2. [An injection, stage by stage](an-injection-stage-by-stage.md). One ordinary finding,
   command injection, followed through every stage, showing what each stage can do with
   it and what a report ends up saying. 
3. [Recall is not evidence](recall-is-not-evidence.md). A check that asks
   whether a named vulnerability is even present before anything is built or run, and
   what gets measured to tell whether it is working.
